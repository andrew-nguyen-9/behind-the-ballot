# Logo brief — Behind the Ballot

> For **Claude Design MCP**: brainstorm variants, import the `.dc.html` (metrotrack
> pattern) `[C2a]`. Cite `BTB_QUESTIONNAIRE.md` `[id]`.

## Direction `[C1a]`
A mark from **civic iconography** — ballot / check / ballot-box. Not a wordmark-only, not
a map-fragment, not abstract-data. The "behind" in the name suggests looking *into* the
ballot — a check mark emerging from / behind a ballot slot is a strong candidate.

## Tone
Non-partisan, trustworthy, editorial (NYT/538 register `[C11a]`). Neutral civic palette —
**no party blue/red in the logo** `[C4a]`; ink + the one brand green (`#0b6e4f`) only.

## Chosen mark (Brand v1.0)
**The transparent ballot box** — an open box, centered slot, a ballot mid-insertion, and a
quiet lattice of data nodes behind the front face ("the intelligence beneath every vote").
Pairs with **Fraunces** wordmark (`Behind the` 400 / `Ballot` 600). Favicon + icon set
generated from `apps/web/public/favicon.svg`.

## Constraints
- Works mono (favicon 16px) and full-color; legible at 16px and on dark `[C6a]`.
- Pairs with the UI sans `[C7a]`.
- Accessible contrast on light + dark backgrounds `[W1a]`.

## Assets to produce `[C3a+C]`
Header lockup, footer mark, favicon (SVG + ICO), default OG image, app icon (maskable).
Per-module marks are a nice-to-have, not V1-blocking.

## Deliverable
Brainstormed variants → chosen `.dc.html` imported into the repo → exported SVG set wired
into `v1.0.7-design-tokens` / `v1.0.8-base-layout-seo`.
